Learning plan
AI Engineer Roadmap
12-month path from Python basics to production AI engineering — with AWS, agents, and a real portfolio.

AI engineer learning roadmap — Python foundations, weekly milestones, skill checklist with resources, daily schedule, and project guides using AWS

Weeks 1–4
Phase 0 — Python foundations
Get fluent enough to build, not just read
▶
Week 1
Core syntax + data types
Variables, strings, lists, dicts, loops, conditionals, functions. Work through Automate the Boring Stuff ch. 1–6. Goal: write a script that reads a file and manipulates its content.
Week 2
Files, modules, packages, virtual envs
Reading/writing files, importing libraries, pip install, creating a venv. Set up VS Code + Python extension. These are the daily tools of every Python AI project.
Week 3
HTTP, JSON, and working with APIs
Use the requests library to call a public API. Parse JSON responses. Handle errors. This is 80% of what calling LLM APIs looks like.
Week 4
Classes, async, and essential AI libraries
Basic OOP. Async/await (LLM streaming uses this). Install and explore: openai, anthropic, httpx, pydantic, python-dotenv, FastAPI.
✓ Milestone: comfortable writing Python from scratch
Months 2–3
Phase 1 — API builder
Turn from consumer to builder
▶
Weeks 5–6
First LLM API call + chatbot
Send messages, handle streaming, manage conversation history, understand token limits and pricing. Build a CLI chatbot with a system prompt.
Week 7
Prompt engineering fundamentals
Read promptingguide.ai. Zero-shot, few-shot, chain-of-thought. Get structured JSON back reliably from a model.
Week 8
Function / tool calling
Define tools and let the model decide when to call them. Build a weather bot that calls a real API.
Weeks 9–10
Project 1: Smart document summarizer
PDF in → structured JSON summary out, deployed as a REST endpoint on AWS.
✓ Milestone: shipped and deployed project 1
Weeks 11–12
Embeddings + fast.ai lesson 1
What embeddings are, how to generate them, cosine similarity. Start fast.ai at 1 lesson/week pace.
Months 4–6
Phase 2 — Agentic systems
The hottest skill in AI engineering right now
▶
Weeks 13–14
RAG fundamentals
Chunk → embed → store in ChromaDB → retrieve → generate. Build a "chat with your docs" app.
Weeks 15–16
LangChain + LangGraph basics
Chains, prompt templates, memory. Build a stateful multi-step pipeline. Learn the concepts more than the library.
Weeks 17–18
Agents + tool use
Agent using web search, calculator, and file reader in a loop. ReAct pattern. Handle tool errors and infinite loops.
Weeks 19–22
Project 2: Research agent
Autonomous agent that searches, reads, synthesises a structured report. Planner + critic for multi-agent stretch goal.
✓ Milestone: shipped project 2
Weeks 23–24
Evaluation frameworks
LLM-as-judge, automated output tests, LangSmith tracing. Evals separate serious AI engineers from hobbyists.
Months 7–9
Phase 3 — Open-source + deployment
Your infra background carries you here
▶
Weeks 25–26
Local models with Ollama
Run Llama 3 or Mistral locally. Compare quality vs GPT-4. Understand quantization and when self-hosting beats an API.
Weeks 27–28
Hugging Face ecosystem
Browse the hub, understand model cards, run inference with Transformers, read benchmark scores meaningfully.
Weeks 29–30
Fine-tuning basics
LoRA/QLoRA on a small model on Google Colab (free GPU). Know when fine-tuning beats better prompting.
Weeks 31–36
Project 3: Production AI app on AWS
Containerise your research agent, deploy to AWS (ECS + ECR or App Runner), real frontend, logging, cost tracking, eval suite.
✓ Milestone: live production-grade AI app
Months 10–12
Phase 4 — Job-ready
Portfolio, positioning, applications
▶
Weeks 37–42
MLOps + polish
Logging, cost tracking, model versioning, evals. Polish READMEs with architecture diagrams. Write 2–3 blog posts about what you built.
Weeks 43–48
Apply + interview prep
Target AI-focused startups. Lean on your Klarna backend+infra credibility. Prep system design: design a RAG pipeline, design an agent orchestration system.
✓ Milestone: land your first AI engineering role

Project 1 Months 2–3
Smart document summarizer
CLI tool → REST API → deployed on AWS
▶
Outcome
A REST API that accepts a PDF, chunks it, processes each chunk through an LLM, and returns a structured JSON summary with key points, entities, and a TL;DR. Deployed to AWS with a real URL you can share.
Stack
PythonFastAPIOpenAI APIPyPDF2PydanticDockerAWS App RunnerAWS ECR
What you'll learn
—Context window limits: a PDF is too big to send at once — you must chunk it
—Prompt design for extraction tasks: getting reliable structured JSON out
—Wrapping an AI feature in a FastAPI endpoint (your backend muscle applies directly)
—Pushing a Docker image to AWS ECR and deploying with App Runner — the simplest AWS deploy path
Stretch goals
Simple HTML frontend. Multiple file types. Caching so the same doc isn't re-processed. Rate limiting.
Project 2 Months 4–6
Research agent
Multi-step agent with web search + synthesis
▶
Outcome
Give it a topic and it autonomously searches the web, reads pages, cross-references sources, and outputs a structured report with citations. Fully autonomous — no hand-holding.
Stack
PythonLangGraphTavily Search APIChromaDBOpenAI APIStreamlit
What you'll learn
—How to build a stateful agent loop, not just a one-shot prompt
—Tool use: search, scraping, and memory tools all working together
—How agents fail (infinite loops, hallucinated tool calls) and how to guard against it
—RAG in practice: agent retrieves from its own memory of previously read pages
Stretch goals
Add a planner agent that breaks the query into sub-tasks, and a critic agent that reviews the output for gaps. Now it's a full multi-agent system.
Project 3 Months 7–9
Production AI app on AWS
Containerised, monitored, live on AWS
▶
Outcome
The research agent, fully productionised: auth, real frontend, Docker, deployed on AWS ECS with an ALB, CloudWatch logging, cost tracking, and a basic eval suite. The one you demo in interviews with a live URL.
Stack
Python / FastAPINext.js or SvelteKitDocker ComposeAWS ECS + ECRAWS ALBCloudWatchRDS Postgres + pgvectorLangSmithOllama
What you'll learn
—ECS + ECR: the standard AWS way to run containerised apps — your infra background applies directly
—CloudWatch for AI observability: tracing a 10-step agent run when something breaks in prod
—Swapping between OpenAI and a local Ollama model via a config flag
—RDS + pgvector: use the Postgres you already know as a vector store — no new DB needed
AWS resources
ECS getting started (free) ECR + Docker push (free) App Runner (simpler alt) (free) AWS free tier (free)
Why this one matters
Most AI engineers can prototype. Very few can ship. A live URL on AWS plus a clean architecture diagram puts you in a different tier of candidate — especially given your existing infra credibility.
